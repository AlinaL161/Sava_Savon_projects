## Описание проекта

*   **Цель проекта:** Провести исследовательское тестирование функциональности «Работа с рецептами» (калькулятор мыла, создание, редактирование, сохранение, публикация, экспорт рецептов) и оценить готовность модуля к релизу.
*   **Объект тестирования:** Веб-приложение калькулятора мыла Savon, включающее модули: создание/редактирование рецепта, калькулятор ингредиентов (масла, кислоты, добавки), сохранение и публикация рецептов, экспорт в JPG/PDF, работа с чужими рецептами.
*   **Объём тестирования:** Выполнено **более 450 проверок** (на основе тестового листа Excel), составлено **33 баг-репорта**.

## Результаты тестирования

### Общая оценка стабильности

Функциональность калькулятора в целом работает, большинство позитивных и негативных сценариев подтверждены (статус **PASSED**).

**Корректно работают:**
- ограничения полей (название ≤40, описание ≤1000 символов);
- валидация спецсимволов, разных алфавитов, пробелов;
- выбор единиц измерения (граммы/унции), процента пережира (1–20%), типа щелочи (NaOH/KOH/смесь);
- добавление масел, кислот, собственных ингредиентов;
- переключение режимов ввода (граммы/проценты) для масел;
- достижение лимита рецептов на бесплатном тарифе (5 рецептов).

### Выявленные дефекты

Выявлен **ряд критических дефектов**, нарушающих бизнес-логику сохранения, редактирования и публикации рецептов. Часть тестов не выполнена (статус «-») или заблокирована из-за найденных ошибок.

**Ключевые критические дефекты:**

| ID | Описание | Влияние |
|----|----------|---------|
| **Баг-3** | Сохраняется рецепт с пустым параметром «пережир» | Нарушение обязательности поля |
| **Баг-5** | Сохраняется рецепт с пустым параметром «расчёт жидкости» | Нарушение обязательности поля |
| **Баг-22** | Сохраняется рецепт без указания веса масла, кислоты или ингредиента | Сохранение неполных данных |
| **Баг-23** | Сохраняется рецепт при общем проценте масел ≠100% | Некорректный расчёт состава |
| **Баг-28** | Создаётся дубль рецепта через «создать на основе» без изменений | Избыточные копии |
| **Баг-29** | Кнопка «редактировать» доступна для рецепта на модерации | Нарушение статусной модели |
| **Баг-30** | В списке «Мои рецепты» неверный статус у рецепта на модерации | Неверное отображение статуса |

**Дефекты, влияющие на удобство и экспорт:**
- **Баг-1, Баг-12** – длинные названия и ингредиенты не переносятся, текст выходит за границы формы.
- **Баг-2, Баг-18, Баг-19** – описание не дублируется в блоке «Мой рецепт» и отсутствует в скачанном файле, строки обрезаются.

**Проблемы синхронизации режимов ввода:**
- **Баг-8** – при переключении режима масел с процентов на граммы данные сбрасываются.
- **Баг-10, Баг-14** – режим ввода для кислот и ингредиентов не синхронизируется между элементами.

### Что работает хорошо

- ✅ Создание, редактирование и удаление черновика авторизованным пользователем.
- ✅ Сохранение рецепта без описания через промежуточное подтверждение.
- ✅ Невозможность сохранить рецепт без названия и без масла (хотя сообщение об ошибке требует доработки).
- ✅ Блок свойств мыла (подсветка значений в диапазоне/вне диапазона).
- ✅ Скачивание рецепта в JPG и PDF (файлы создаются, но содержимое требует доработки).
- ✅ Сценарий «создать на основе» чужого публичного рецепта.

### Непокрытые и неоднозначные моменты

- Поведение при сохранении с общим процентом масел ≠100% (сейчас ошибочно сохраняется).
- Расчёт процента кислот и дополнительных ингредиентов – от чего считается?
- Что происходит при сохранении без указания веса кислоты/ингредиента.
- Корректность названия строки «Масса добавок» (Баг-17 – названо «Общая масса»).
- Сценарии после модерации (одобрено/отклонено) – требуется проверка через телеграм-бота.
- Сохранение данных после обновления страницы (Баг-31 – введённые данные сбрасываются).

## Итоговая оценка и рекомендации

**Функциональность «Работа с рецептами» находится в стадии «требуется доработка перед релизом».**

Базовые сценарии стабильны, но наличие **критических дефектов**, связанных с сохранением неполных данных и некорректной публикацией, делает невозможным выпуск без исправлений.

**Рекомендации:**

1. **Исправить критические дефекты сохранения** (Баг-3, -5, -22, -23) – без этого пользователь может создавать невалидные рецепты, нарушающие логику калькулятора.
2. **Доработать статусную модель и публикацию** (Баг-28, -29, -30) – рецепт на модерации не должен редактироваться и должен корректно отображать свой статус.
3. **Улучшить UX отображения и экспорта** (Баг-1, -2, -12, -18, -19) – длинные тексты должны переноситься, описание – присутствовать в рецепте и скачанном файле.
4. **Синхронизировать режимы ввода** для кислот и ингредиентов (Баг-10, -14).
5. **Провести дотестирование** незавершённых сценариев (модерация, восстановление данных после обновления страницы).

**Итоговая рекомендация:** **Продукт к релизу не рекомендуется** до полного устранения найденных критических дефектов и проведения повторного тестирования (ретеста).
